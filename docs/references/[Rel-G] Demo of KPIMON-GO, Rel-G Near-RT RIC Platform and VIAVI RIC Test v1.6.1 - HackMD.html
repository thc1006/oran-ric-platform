3. Execute!

3.1 RIC Test: Run the Scenario Generator

Execution:
Go to Runtime -> Test Control.

Press Run test.

Check:
Go to Reports -> E2 Message Flows.

Press ASN1-dump-for gNB-123456.

Note:
Please check the E2 Setup Procedure has passed successfully. (Receive the E2Setup Response Message)

If no, please debug the connection between RIC Test and E2Term or between E2Term and E2mgr.

If pass successfully, you can go to next step Execute KPIMON-GO xApp.

3.2 Near-RT RIC: Execute KPIMON-GO xApp

Install xApp:







# Onboard xApp Configuration
cd kpimon-go/deploy
dms_cli onboard config.json schema.json

# Deploy xApp
dms_cli install kpimon-go 2.0.1 ricxapp

Check Log:
Here, you can check the log for E2Term and KPIMON-GO xApp. Even, you can capture the packet between RIC Test and E2Term.

If xApp has already installed. You only need to do this:

kubectl delete pods -n ricxapp $(kubectl get pods -n ricxapp | grep kpimon | awk '{print $1}')
